# -CS-255-System-Anaylsis-and-Design

#Briefly summarize the DriverPass project. Who was the client? What type of system did they want you to design?
The DriverPass project was for a client who wanted a web-based system to help students prepare for driving tests through online practice exams, progress tracking, and scheduling in-person driving lessons. The system was designed to support customers, instructors, and administrators by managing accounts, appointments, training materials, and business data.

#What did you do particularly well?
I did particularly well at identifying the system requirements and translating the client’s needs into functional and technical features. My documents clearly outlined user roles, security requirements, scheduling functions, progress tracking, and the system components needed for the DriverPass application.

#If you could choose one part of your work on these documents to revise, what would you pick? How would you improve it?
I would revise the UML diagrams in the System Design Document by adding more detail and ensuring every interaction between users and system components was fully represented. I would improve them by reviewing the requirements again and making sure each major function had a clear connection within the diagrams.

#How did you interpret the user’s needs and implement them into your system design? Why is it so important to consider the user’s needs when designing?
I interpreted the user’s needs by focusing on the problems DriverPass wanted to solve, such as helping students pass driving tests and providing easier access to training resources. Considering user needs is important because a system must be practical, easy to use, and provide the features that solve the actual problems users experience.

#How do you approach designing software? What techniques or strategies would you use in the future to analyze and design a system?
I approach software design by first understanding the user requirements, identifying system functions, and then creating models that show how the system will operate. In the future, I would continue using techniques such as requirements gathering, UML diagrams, user feedback, and iterative improvements to create systems that are efficient and user-focused.
